# Docs Sync Automation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automatically open a PR in `veecode-platform/docs` whenever `veecode-platform/devportal-platform` publishes a release, with Claude analyzing the diff and generating surgical doc patches.

**Architecture:** A GitHub Actions workflow in `devportal-platform` triggers on `release: published`, builds a prompt from the release diff and the affected doc files (identified via `dp-source` anchor comments), runs `claude -p` in headless mode, parses the structured output, and opens a PR in the docs repo. Human review is required before merge.

**Tech Stack:** GitHub Actions, `claude -p` (Claude Code CLI, headless), Python 3 (stdlib only), `patch(1)`, `gh` CLI, `git`.

## Global Constraints

- `claude -p` invocation requires `ANTHROPIC_API_KEY` in the environment.
- Cross-repo operations require `DOCS_REPO_TOKEN` (GitHub PAT with `repo` scope) in `devportal-platform` secrets.
- Claude model: `claude-sonnet-4-6`.
- PRs **never** auto-merge — always require human approval.
- Domain vocabulary (anchor keywords): `storage`, `pvc`, `postgres`, `plugin`, `entrypoint`, `preset`, `helm`, `env`. Additions must be reflected in both the grep command (workflow) and the doc anchor.
- Anchor format: `<!-- dp-source: domain1,domain2 -->` — lowercase, comma-separated, no spaces after commas.
- All workflow output (`/tmp/*.txt`, `/tmp/*.patch`) is ephemeral to the runner — no secrets written to disk.
- Tasks 1 runs in `veecode-platform/docs`. Tasks 2–4 run in `veecode-platform/devportal-platform`.

- [ ] **Step 1: Create `.github/prompts/` directory and the template file**

```bash

```

Create `.github/prompts/sync-docs.md` with this exact content:

````markdown
You are a surgical technical writer. Your sole job is to keep the Docusaurus
documentation at veecode-platform/docs accurate after a devportal-platform release.
You never invent features. You never remove documented behavior unless the diff
explicitly removes it. You only fix what the diff makes demonstrably wrong.

---

## Release context
Release tag: {RELEASE_TAG}
Previous tag: {PREV_TAG}

## What changed (git diff {PREV_TAG}..{RELEASE_TAG})
```diff
{GIT_DIFF}
```

## Documentation files to review
These files carry `dp-source` anchors that overlap with domains changed in this
release. Review each for stale claims.

{FILE_SECTIONS}

---

## Your task — follow these steps in order

**Step 1 — Extract semantic changes**
List only behavioral/architectural changes a DevPortal operator would care about.
Exclude: code style, test changes, CI changes, variable renames with no behavioral
effect. Examples of semantic changes: a new required env var, a removed volume,
a changed default, a new dependency, a replaced storage backend.

**Step 2 — Map to documentation**
For each semantic change from Step 1, find the exact sentence(s) in the provided
files that describe the now-stale behavior. Quote them verbatim. If no sentence
is stale for a given change, say "no stale coverage found".

**Step 3 — Write minimal patches**
For each stale sentence from Step 2, write a unified diff hunk that corrects only
that sentence. Do not touch surrounding text. Do not add new sections unless the
diff introduces a feature with zero existing doc coverage.

**Step 4 — Verify before output**
For each hunk you wrote, silently confirm: "this change is justified by [specific
lines in the diff]". If you cannot name the justification, remove the hunk.

---

## Output format — respond in exactly this structure, nothing else

### ANALYSIS
[Bullet list of semantic changes from Step 1. If none: "No semantic changes found."]

### PATCHES
[One block per changed file. Omit this section entirely if no patches.]

#### File: {relative/path/from/repo/root.md}
```diff
--- a/{relative/path/from/repo/root.md}
+++ b/{relative/path/from/repo/root.md}
@@ -{line},{count} +{line},{count} @@
 context line
-stale line
+corrected line
 context line
```

### PR_BODY
[2–4 sentences: what the release changed, which docs were updated, and why.
Plain English. Suitable for a GitHub PR description.]

### UNCERTAIN
[Sections where you spotted a possible issue but lacked confidence to patch.
Format: "path/to/file.md, line ~N: what concerns you"
If none: "None."]

---

## Hard constraints
- Patch ONLY lines directly contradicted by the diff.
- Never add paragraphs, sections, or feature descriptions not present in the diff.
- Never remove documented behavior unless the diff explicitly removes it.
- If a section seems stale but the diff doesn't confirm it, put it in UNCERTAIN.
- If nothing needs changing: output only the single line `NO_CHANGES_NEEDED`.
````

## Task 3: Create the sync-docs workflow

**Repo:** `veecode-platform/devportal-platform`

**Files:**
- Create: `.github/workflows/sync-docs.yml`

**Interfaces:**
- Consumes: `.github/prompts/sync-docs.md` (Task 2).
- Consumes: `dp-source` anchors in `veecode-platform/docs` (Task 1).
- Produces: a PR in `veecode-platform/docs` with label `auto-generated/docs-sync`.

- [ ] **Step 1: Create `.github/workflows/sync-docs.yml`**

```yaml
name: Sync docs on release

on:
  release:
    types: [published]
  workflow_dispatch:          # manual trigger for testing
    inputs:
      release_tag:
        description: 'Release tag to simulate (e.g. v2.2.1)'
        required: true
      prev_tag:
        description: 'Previous tag to diff against (e.g. v2.2.0)'
        required: true

jobs:
  sync-docs:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout devportal-platform
        uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Resolve tags
        id: tags
        run: |
          if [ "${{ github.event_name }}" = "workflow_dispatch" ]; then
            RELEASE_TAG="${{ github.event.inputs.release_tag }}"
            PREV_TAG="${{ github.event.inputs.prev_tag }}"
          else
            RELEASE_TAG="${{ github.event.release.tag_name }}"
            PREV_TAG=$(git describe --tags --abbrev=0 "${RELEASE_TAG}^" 2>/dev/null || echo "")
          fi

          if [ -z "$PREV_TAG" ]; then
            echo "No previous tag found — cannot compute diff. Skipping."
            exit 0
          fi

          echo "release_tag=$RELEASE_TAG" >> $GITHUB_OUTPUT
          echo "prev_tag=$PREV_TAG"       >> $GITHUB_OUTPUT

      - name: Compute release diff
        id: diff
        run: |
          git diff "${{ steps.tags.outputs.prev_tag }}..${{ steps.tags.outputs.release_tag }}" \
            > /tmp/release.diff
          echo "Diff size: $(wc -c < /tmp/release.diff) bytes"

      - name: Extract affected domains from diff
        id: domains
        run: |
          DOMAINS=$(grep -oiE '\b(pvc|postgres|plugin|storage|entrypoint|preset|helm|env)\b' \
            /tmp/release.diff | tr '[:upper:]' '[:lower:]' | sort -u | tr '\n' '|' | sed 's/|$//')

          if [ -z "$DOMAINS" ]; then
            echo "No recognized domains in diff — nothing to do."
            echo "skip=true" >> $GITHUB_OUTPUT
          else
            echo "Domains found: $DOMAINS"
            echo "domains=$DOMAINS" >> $GITHUB_OUTPUT
          fi

      - name: Checkout docs repo
        if: steps.domains.outputs.skip != 'true'
        uses: actions/checkout@v4
        with:
          repository: veecode-platform/docs
          token: ${{ secrets.DOCS_REPO_TOKEN }}
          path: docs-repo

      - name: Find affected doc files via dp-source anchors
        id: anchors
        if: steps.domains.outputs.skip != 'true'
        run: |
          DOMAINS="${{ steps.domains.outputs.domains }}"
          # Build grep ERE pattern: pvc|postgres|...
          PATTERN="dp-source:.*($DOMAINS)"

          grep -rl "$PATTERN" docs-repo/devportal/ > /tmp/affected_files.txt 2>/dev/null || true

          if [ ! -s /tmp/affected_files.txt ]; then
            echo "No doc files matched domains '$DOMAINS' — nothing to do."
            echo "skip=true" >> $GITHUB_OUTPUT
          else
            echo "Affected files:"
            cat /tmp/affected_files.txt
          fi

      - name: Build Claude prompt
        if: steps.anchors.outputs.skip != 'true'
        run: |
          python3 - <<'PYEOF'
          import os, pathlib

          template     = pathlib.Path('.github/prompts/sync-docs.md').read_text()
          diff_content = pathlib.Path('/tmp/release.diff').read_text()
          affected     = [l.strip() for l in pathlib.Path('/tmp/affected_files.txt').read_text().splitlines() if l.strip()]

          sections = []
          for fpath in affected:
              p = pathlib.Path(fpath)
              if p.exists():
                  rel = str(p.relative_to('docs-repo'))
                  sections.append(f"### {rel}\n```\n{p.read_text()}\n```")
          file_sections = "\n\n".join(sections)

          rendered = (template
              .replace('{RELEASE_TAG}',  os.environ['RELEASE_TAG'])
              .replace('{PREV_TAG}',     os.environ['PREV_TAG'])
              .replace('{GIT_DIFF}',     diff_content)
              .replace('{FILE_SECTIONS}', file_sections))

          pathlib.Path('/tmp/final-prompt.txt').write_text(rendered)
          print(f"Prompt built: {len(rendered)} chars, {len(affected)} file(s)")
          PYEOF
        env:
          RELEASE_TAG: ${{ steps.tags.outputs.release_tag }}
          PREV_TAG:    ${{ steps.tags.outputs.prev_tag }}

      - name: Install Claude CLI
        if: steps.anchors.outputs.skip != 'true'
        run: npm install -g @anthropic-ai/claude-code

      - name: Run Claude (headless)
        if: steps.anchors.outputs.skip != 'true'
        run: |
          claude -p "$(cat /tmp/final-prompt.txt)" \
            --model claude-sonnet-4-6 \
            --output-format text \
            > /tmp/claude-output.txt
          echo "Claude output: $(wc -l < /tmp/claude-output.txt) lines"
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}

      - name: Parse Claude output and apply patches
        id: patch
        if: steps.anchors.outputs.skip != 'true'
        run: |
          python3 - <<'PYEOF'
          import re, pathlib, subprocess, sys

          out = pathlib.Path('/tmp/claude-output.txt').read_text()

          if out.strip() == 'NO_CHANGES_NEEDED':
              print("Claude determined no changes are needed.")
              sys.exit(0)

          pr_body_m = re.search(r'### PR_BODY\n(.*?)(?=### UNCERTAIN|$)', out, re.DOTALL)
          patches_m = re.search(r'### PATCHES\n(.*?)(?=### PR_BODY|$)', out, re.DOTALL)

          pr_body = pr_body_m.group(1).strip() if pr_body_m else "Docs sync from devportal-platform release."
          pathlib.Path('/tmp/pr-body.txt').write_text(pr_body)

          if not patches_m or not patches_m.group(1).strip():
              print("No PATCHES section in output — nothing to apply.")
              sys.exit(0)

          patches_text = patches_m.group(1)
          applied = 0
          for block in re.finditer(
              r'#### File: (.+?)\n```diff\n(.*?)```',
              patches_text, re.DOTALL
          ):
              rel_path  = block.group(1).strip()
              diff_text = block.group(2)
              patch_file = f'/tmp/patch_{rel_path.replace("/", "_")}.patch'
              pathlib.Path(patch_file).write_text(diff_text)
              result = subprocess.run(
                  ['patch', '-p1', '--input', patch_file],
                  cwd='docs-repo', capture_output=True, text=True
              )
              if result.returncode != 0:
                  print(f"WARN: patch failed for {rel_path}: {result.stderr}")
              else:
                  print(f"Applied patch: {rel_path}")
                  applied += 1

          if applied == 0:
              print("No patches applied.")
          else:
              # Signal to next step that we have changes
              pathlib.Path('/tmp/has_changes').write_text('true')
          PYEOF

      - name: Commit and open PR in docs repo
        if: steps.anchors.outputs.skip != 'true'
        run: |
          [ -f /tmp/has_changes ] || { echo "Nothing to commit."; exit 0; }

          cd docs-repo
          BRANCH="docs-sync/${{ steps.tags.outputs.release_tag }}"

          git config user.name  "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git checkout -b "$BRANCH"
          git add -A
          git diff --cached --quiet && { echo "No file changes after patch."; exit 0; }
          git commit -m "docs: sync with devportal-platform ${{ steps.tags.outputs.release_tag }}"
          git push origin "$BRANCH"

          gh pr create \
            --title "docs: sync with release ${{ steps.tags.outputs.release_tag }}" \
            --body-file /tmp/pr-body.txt \
            --label "auto-generated/docs-sync" \
            --repo veecode-platform/docs
        env:
          GH_TOKEN: ${{ secrets.DOCS_REPO_TOKEN }}
```
